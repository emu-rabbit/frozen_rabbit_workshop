# 核心原則 (請先閱讀)

> **⚠️ 絕對禁止觸碰已完成的筆記**
>
> `name` 欄位若已是包含 `tw`、`cn`、`en`、`ja` 四個欄位的物件，代表該筆記已由人工或前次 workflow 確認完畢。
> **任何情況下都不得修改已完成物件的任何欄位，即使你認為翻譯有誤也不行。**
> 如需修改既有翻譯，使用者必須先手動將該筆記的 `name` 改回字串形式，workflow 才能重新處理。

---

# 執行步驟

## Step 1：掃描並分類

掃描 `src/data/recommended/` 目錄下的所有 `.json` 檔案，對每個檔案中的每個筆記判斷狀態：

- **待處理 (Pending)**：`name` 是字串 → 需要展開為多語言物件
- **已完成 (Done)**：`name` 已是物件（含 `tw`, `cn`, `en`, `ja`）→ **完全跳過，不得修改**

只對「待處理」的筆記執行後續步驟。若所有筆記皆已完成，直接跳到 Step 5。

## Step 2：JSON 語法修正

對有待處理筆記的檔案，檢查並修正 JSON 格式錯誤（如多個物件連排）。

## Step 3：翻譯待處理筆記

對每筆 `name` 為字串的筆記，依照以下翻譯規則將其展開為物件：

```json
{
  "tw": "（原始繁體中文字串）",
  "cn": "（簡體中文翻譯）",
  "en": "（英文翻譯）",
  "ja": "（日文翻譯）"
}
```

### 翻譯關鍵字對照表

| 關鍵字 (TW) | 簡體中文 (cn) | 英文 (en) | 日文 (ja) |
| :--- | :--- | :--- | :--- |
| **禦敵** | 御敌 | Fending | タンク |
| **制敵** | 制敌 | Maiming | メレー |
| **強襲** | 强袭 | Striking | メレー |
| **游擊** | 游击 | Scouting | メレー |
| **精準** | 精准 | Aiming | レンジ |
| **詠咒** | 咏咒 | Casting | キャス |
| **治癒** | 治愈 | Healing | ヒラ |
| **大地** | 大地 | Gathering | ギャザラー |
| **巧匠** | 巧匠 | Crafting | クラフター |
| **工具** | 工具 | Tools | 道具 |
| **{n}件套** | {n}件套 | {n}-piece Set | {n}点セット |
| **{n}件** | {n}件 | {n}-piece | {n}点 |

### 翻譯原則

- **關鍵字一致性 (Search Stability)**：表格中的關鍵字必須嚴格遵守，不得自行調整翻譯。
- **上下文智慧翻譯 (AI Contextual Intelligence)**：表格未涵蓋的詞彙（如裝備部位名稱等），請結合 FFXIV 遊戲背景，提供最自然的翻譯。
- **完備性要求 (Zero-Leakage)**：`en`、`ja`、`cn` 欄位中絕不能殘留繁體中文字符。
- **數字規則**：`cn` 欄位沿用繁體中文原本的數字形式（如「十件套」），`en`/`ja` 使用阿拉伯數字（如「10-piece Set」、「10点セット」）。
- **空格規則**：英文翻譯中，職業關鍵字與後續詞彙之間保留空格（`Fending 10-piece Set`，而非 `Fending10-piece Set`）。

## Step 4：寫回檔案

使用 `write_to_file` 工具將修改後的 JSON 寫回原始檔案，格式需保持一致（2 space indent）。

> **⚠️ 嚴禁使用 PowerShell 的 `Set-Content`、`>` 或 `Out-File`**，這些指令會將編碼改為 UTF-16，導致中日文字符變成亂碼。

## Step 5：更新 index.ts

若有任何新檔案被新增或舊檔案被修正，才需要更新 `src/data/recommended/index.ts`。

- 使用 `import type { Note }` 格式（不得使用 `import { Note }`）
- 使用 `export default allRecommendedNotes` 格式
- 陣列中的筆記順序以 **iLv 由小到大** 為準（可在 `index.ts` 的陣列宣告順序中控制，或加入 `.sort()` 動態排序）

## Step 6：回報結果

處理完成後，向使用者回報：
- ✅ **已處理**：列出每筆被展開翻譯的筆記名稱與翻譯結果
- ⏭️ **已跳過**：列出「因已完成而跳過」的筆記數量（不需列出名稱）
- ⚠️ **警告**：任何無法處理的檔案或翻譯疑慮

# 新增筆記的標準流程

當你（或使用者）想新增一筆推薦筆記時，請依照以下步驟：

1. 在對應的 `.json` 檔案末尾新增一個物件，`name` 欄位**以字串形式**填入繁體中文名稱：
   ```json
   {
     "id": "（新的 UUID）",
     "name": "Lv.100 iLv750 巧匠主副手16件套",
     "items": [...],
     "createdAt": "（ISO 8601 時間戳）"
   }
   ```
2. 執行 `/process-recommended` workflow，由 agent 自動展開翻譯。
3. 確認翻譯結果後，執行 `/add-commit-all` 提交。
