# 執行步驟

1. 掃描 `src/data/recommended/` 目錄下的所有 `.json` 檔案。
2. 對於每個檔案，修正所有對於 JSON 格式來說是錯誤的語法，使其成為正確可以解析的 JSON 檔。
3. 尋找並分析每個檔案中所有的推薦筆記 (`Note` 物件)，特別檢查其 `name` 屬性。
4. 如果 `name` 是字串，請將他展開成物件。如果 `name` 已經是物件且包含 `tw` 屬性，則分析其 `tw` 內容。
5. 由於本筆記為 Final Fantasy XIV (FF14) 遊戲相關工具，對於裝備套裝名稱，請務必辨識名稱中的關鍵字並遵循以下對應規則進行翻譯，不得自行調整：

| 關鍵字 (TW) | 簡體中文 (cn) | 英文 (en) | 日文 (ja) |
| :--- | :--- | :--- | :--- |
| **禦敵** | 御敌 | Fending | タンク |
| **制敵** | 制敌 | Maiming | メレー |
| **強襲** | 强袭 | Striking | メレー |
| **游擊** | 游击 | Scouting | メレー |
| **精準** | 精准 | Aiming | レンジ |
| **詠咒** | 咏咒 | Casting | キャス |
| **治癒** | 治愈 | Healing | ヒラ |
| **大地** | 大地 | Gathering | ギャザラー |
| **巧匠** | 巧匠 | Crafting | クラフター |

**核心翻譯原則：**
- **關鍵字一致性 (Search Stability)**：翻譯表中的關鍵字是為了確保使用者能透過統一代稱在不同語言下搜尋到對應資料。對於表中的字詞，請務必嚴格遵守。
- **上下文智慧翻譯 (AI Contextual Intelligence)**：翻譯表並非唯一的依歸。對於未在表中的詞彙（如「衣服」、「件套裝」、「例外的名稱」等），請發揮 AI 的語言理解能力，結合 FFXIV 遊戲背景，提供該語系下最自然、正確的翻譯。
- **完備性要求 (Zero-Leakage)**：請確保 `en`、`ja`、`cn` 欄位中絕不殘留任何原始的繁體中文字符。即便該字詞不在翻譯表中，也應根據上下文進行智慧轉譯（例如將「件套裝」轉譯為英文的「-piece Set」或日文的「点セット」）。

6. **開發者防呆與環境規範 (Safeguards)：**
    - **結構修補**：部分損壞檔案可能呈現多個 JSON 物件連續排列（如 `{...}{...}`）。處理時應將其包裹於 `[]` 中並在物件間補上 `,`。
    - **編碼絕對保護**：**嚴禁使用 PowerShell 的 `Set-Content`、`>` 或 `Out-File` 等指令批次處理 JSON**。這些指令會強制改變檔案編碼（如轉為 UTF-16），導致繁中/日元字符轉為亂碼。
    - **推薦處理方式**：建議撰寫獨立的 Node.js 腳本（使用 `fs.writeFileSync(path, data, 'utf8')`）或使用 `write_to_file` 工具進行純 UTF-8 寫入。
    - **格式微調**：英文翻譯中，職業關鍵字與數量之間應保留空格（例如：`Fending 10-piece Set`，而非 `Fending10-piece Set`）。

7. 將處理後的內容寫回各自原始的 JSON 檔案中，請務必保持原本的 JSON 縮排與格式。
8. 更新 `src/data/recommended/index.ts`：確保所有處理過的檔案（尤其是新修復或原本遺漏的）皆已正確匯入並加入至 `allRecommendedNotes` 陣列中。
9. 最後請確保所有檔案中每個物件的格式都符合 `src/types/note.ts` 中 `Note` 的資料格式定義。
10. 處理完成後，請整理一份清單向使用者回報你處理了哪些檔案、翻譯了哪些筆記以及其結果。
