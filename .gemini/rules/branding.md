# 🐰 冷凍兔肉的工坊 - 品牌氛圍與 UI/UX 規範

本規範旨在維持專案整體的溫馨、可愛且具工坊代入感的體驗。所有開發人員（包含 AI Agent）必須遵守：

## 1. 用語與氣氛 (Atmosphere)
- **拒絕冷硬名詞**：嚴禁使用「分析」、「決策中心」、「資產管理」等極具現代企業感的詞彙。
- **統一溫馨用詞**：優先使用與「工坊」、「筆記」、「備料」相關的柔和名詞。
  - 核心工作區 ➡️ **「備料台」** (Prep Table)
  - 物品清單 ➡️ **「筆記」** (Note) / **「清單」** (List)
  - 推薦功能 ➡️ **「兔肉私心筆記」**
  - 使用者收藏 ➡️ **「收藏的小筆記」**
  - 歷史紀錄 ➡️ **「翻開舊紀錄」**
- **一致性**：確保在 UI 字串、API 欄位說明（若可見）以及多語系字典中，對同一個功能的稱呼保持絕對一致。

## 2. UI 佈局原則
- **側邊欄打平**：側邊欄導覽必須維持單層扁平結構，禁止使用巢狀選單 (Nested menus)。
- **導航順序**：
  1. `寫張新筆記` (置頂功能)
  2. `收藏的小筆記`
  3. `兔肉私心筆記`
  4. `翻開舊紀錄`
  5. `設定` (置於底部)
- **排版穩定性**：列表式 UI 的操作按鈕（如：刪除、跳轉）必須設定固定寬度且禁止縮放 (`shrink-0`)，確保在不同長度的內容下介面始終維持穩定。
- **手動斷行**：支持在多多語系字串中透過 `<br />` 標籤手動控制換行位置（需配 `v-html` 使用）。

## 3. 資料與多語系策略 (i18n)
- **強健備援 (Fallback)**：當目標語系字典缺失 (404) 時，必須自動回退至英文字典，確保物品名稱不會顯示為「未知」。
- **解耦顯示**：圖示 (Icon) 的顯示應與翻譯名稱 (Name) 解耦。只要 ID 存在，就應顯示圖示，不應受翻譯缺失影響。
