# FF14 巧匠輔助工具 - XIVAPI 整合規範

本專案（冷凍兔肉的工坊）將高度密合的與 [XIVAPI](https://xivapi.com/) 整合。在進行開發時，所有開發人員（包含 AI Agent）必須遵守以下規則：

## 1. 物品與資料 ID 使用原則
- **一律使用 XIVAPI 原始 ID**：專案內所有稱呼「物品 ID」、「配方 ID」或「任務 ID」等任何資料關聯欄位，都必須直接對應並相容於 XIVAPI 端提供的整數 ID 或字串識別碼。
- **後端單一事實來源**：應用程式本身不應自行建立或配發獨立的不關連虛擬 ID 系統，以確保未來從 XIVAPI 獲取圖片、名稱、配方成本時的無縫反查。

## 2. API 查詢與快取規劃 (未來實踐方向)
- 處理 ID 輸入或儲存時，要考量到後續異步載入 API 詳細欄位的情況。例如在清單中僅儲存 `[2341, 192, ...]` 等 ID 清單，並在呈現層才去查詢擴充為具有名稱與圖像的物件。

## 核心規範 (General Constraints)
- **語言偏好**：必須優先使用**繁體中文 (Traditional Chinese)** 回應。
- **TODO 限制**：嚴禁擅自動工 `TODO.md`，除非使用者明確要求。
